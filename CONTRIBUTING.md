First off, thanks for taking the time to contribute to ***LoIDE***! :+1:  
It's people like you that make LoIDE such a great tool. 

The following is a set of guidelines for contributing to LoIDE on GitHub.
These are mostly guidelines, not rules.
Use your best judgment, and feel free to propose changes to this document in a pull request.

Following these guidelines helps to communicate that you respect the time of the developers managing and developing this open source project.
In return, they should reciprocate that respect in addressing your issue, assessing changes, and helping you finalize your pull requests.

LoIDE is an open source project and we love to receive contributions from our community — you!
There are many ways to contribute, from writing tutorials or blog posts, improving the documentation, submitting bug reports and feature requests or writing code which can be incorporated into LoIDE itself.
